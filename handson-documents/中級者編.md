## エージェントの構造

> **エージェント ＝ モデル ＋ ハーネス**

```
┌───────────────────────────────────────┐
│             AI Agent                  │
│                                       │
│  ┌─────────┐    ┌────────────────┐   │
│  │  Model  │◀──▶│    Harness     │   │
│  │ (LLM)  │    │                │   │
│  └─────────┘    │  ・コンテキスト管理  │   │
│                 │  ・アーキテクチャ制約  │   │
│                 │  ・ガードレール     │   │
│                 └────────────────┘   │
└───────────────────────────────────────┘
```

LLM だけでは「何でもする」エージェントになる。
ハーネスで「**何をしてよいか**」を設計する。

---

## GitHub Copilot のハーネス構成

| レイヤー | ファイル/ツール | 役割 |
|---------|-------------|------|
| **Instructions** | `.github/instructions/*.md` | 常時適用されるルール |
| **Skills** | `.github/skills/*/SKILL.md` | 特定タスクの手順書 |
| **Hooks** | `.github/hooks/*.json` | ツール実行の制御 |
| **SDK** | `@github/copilot-sdk` | プログラムからの制御 |

---

## ハーネスエンジニアリングの 3 本柱

1. **コンテキスト管理**
   必要な情報をエージェントに与え、不要な情報を与えない

2. **アーキテクチャ制約**
   `allowedTools` で使えるツールを限定する

3. **品質ゲート**
   `SkillTestHarness` のような検証を実施するルールで「意図通りに動くか」を自動検証

## playwright-cli とは

Playwright チームが**コーディングエージェント向けに設計した CLI ツール**。

- トークン効率を最重視したコマンド体系
- GitHub Copilot・Claude Code から直接呼び出せる
- `playwright-cli install --skills` で Copilot 用 Skill が自動生成される(この作業は済)

---

## 基本コマンド

ユーザーがこれをつかってもいいがこれらのコマンドなどを使用してGHCがWebアプリを操作することができる。

```bash
# ブラウザでページを開く
playwright-cli open https://demo.playwright.dev/todomvc/ --headed

# ページ構造のスナップショット取得
playwright-cli snapshot

# スクリーンショット保存
playwright-cli screenshot --filename=temp/todomvc-initial.png

# ネットワーク通信の確認（APIエンドポイント発見）
playwright-cli network
```

スナップショットの出力を「エージェントへのコンテキスト」として使用することもできる

# 5. .NET ダッシュボードアプリの作成

---

## 作成するアプリの仕様

```
src/DashboardApi/
  ├── Program.cs          ← Minimal API エントリポイント
  ├── Models/
  │   └── TodoItem.cs     ← データモデル
  └── Data/
      └── dashboard.db    ← SQLite データストア
```

**API エンドポイント**:

```
GET  /          → ダッシュボード HTML
POST /api/import-csv  → CSV インポート
GET  /api/todos       → 保存済みデータ取得
```

---

## Copilot に作成を依頼する

```markdown
src/DashboardApi に .NET 10 Minimal API のプロジェクトを作成してください。

要件:
- POST /api/import-csv で CSV ファイルを受け取り SQLite に保存する
- GET /api/todos で保存済みデータをJSON形式で返す
- GET / でシンプルな HTML のダッシュボードを返す
- CSV の形式: id,title,completed（ヘッダー行あり）
- レスポンス形式: { success: bool, message: string, count?: int }
- パッケージ: Microsoft.Data.Sqlite
```

---

## Instructions でコード品質を確保する

`.github/instructions/dotnet-api.instructions.md` を作成:

```markdown
---
applyTo: "src/DashboardApi/**/*.cs"
---

# .NET Minimal API 開発ガイドライン

## API 設計
- CSV インポートは POST /api/import-csv に統一する
- レスポンスは必ず { success: bool, message: string, count?: int } 形式
- 全エンドポイントで例外ハンドリングを実装する

## コード品質
- async/await を一貫して使用する
- using ステートメントでリソースを適切に解放する
```

## csv-importer Skill の設計

```markdown
---
name: csv-importer
description: "CSVファイルをダッシュボードAPIにインポートするSkill"
allowed-tools: read_file shell
---

## Instructions

### 手順
1. 指定された CSV ファイルを確認する
2. POST /api/import-csv にアップロードする
   `curl -X POST http://localhost:5000/api/import-csv ...`
3. レスポンスの success フィールドを確認して報告する
```

```markdown
- allowed-tools: read_file shell のみ許可
```

# 7. Copilot SDK バッチアプリの構築

---

## バッチアプリの全体設計

2 つの Skill を順番に実行するバッチ処理:

```
[BatchApp]
    │
    ├─ Session 1: web-data-collector Skill
    │      playwright-cli で ToDoMVC 分析
    │      → temp/collected-data.csv に保存
    │
    └─ Session 2: csv-importer Skill
           CSV を POST /api/import-csv でインポート
           → ダッシュボードにデータ反映
```

---

## Copilot SDK の基本パターン（TypeScript）

```typescript
import { CopilotClient } from "@github/copilot-sdk";

const client = new CopilotClient();
await client.start();

// Skill を指定してセッション作成
const session = await client.createSession({
  model: "claude-sonnet-4.6",
  skillDirectories: [".github/skills/web-data-collector"],
});

// イベントを受け取りながらプロンプト送信
session.on((event) => {
  if (event.type === "assistant.message") process.stdout.write(event.data.content);
  if (event.type === "session.idle") console.log("\n完了");
});

await session.send({ prompt: "ToDoMVC からデータを収集してください" });
```

---

## バッチアプリを作成する

Copilot に依頼:

```markdown
src/BatchApp に TypeScript のバッチアプリを作成してください。

要件:
- @github/copilot-sdk を使用する
- web-data-collector Skill でデータ収集（Session 1）
- csv-importer Skill でインポート（Session 2）
- 各セッションは独立して作成・破棄する
- 実行ログを標準出力に出力する
- 環境変数 BATCH_MODEL でモデルを切り替えられる

実行コマンド: npx ts-node src/BatchApp/index.ts
```

---

## Skill 分離の重要性

```typescript
// ✅ 良い設計：Skill ごとにセッションを分離
const collectSession = await client.createSession({
  skillDirectories: [".github/skills/web-data-collector"],  // playwright-cli のみ
});
// ...収集処理...
await collectSession.destroy();

const importSession = await client.createSession({
  skillDirectories: [".github/skills/csv-importer"],  // shell のみ
});
// ...インポート処理...
await importSession.destroy();
```

→ 1 つのセッションで全部やると制約が混在してしまう

## ハーネスエンジニアリングの本質

```
設計フェーズ:  「AIに何をさせるか」をコードで表現する
              → SKILL.md / allowedTools / AcceptanceCriteria

テストフェーズ: 「AIが意図通りに動くか」を自動検証する
              → SkillTestHarness (mock mode)

運用フェーズ:  「継続的に品質を保つ」CI/CD に統合する
              → GitHub Actions + live mode
```

AI の振る舞いを**通常のソフトウェアと同じように**設計・テスト・管理できる

## 今後の発展

**1. Hooks による高度な制御**
- `PreToolUse` で危険なコマンドをブロック
- `PostToolUse` で実行ログを外部サービスに送信

**2. MCP との連携**
- playwright-cli の代わりに Playwright MCP を使う
- 複雑な対話型ワークフローに適している

**3. 複数 Skill のオーケストレーション**
- 3 つ以上の Skill を組み合わせたパイプライン
- エラー時のロールバック設計
