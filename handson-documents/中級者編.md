## エージェントの構造

> **エージェント ＝ モデル ＋ ハーネス**

```
┌───────────────────────────────────────┐
│             AI Agent                  │
│                                       │
│  ┌─────────┐    ┌────────────────┐   │
│  │  Model  │◀──▶│    Harness     │   │
│  │ (LLM)  │    │                │   │
│  └─────────┘    │  ・コンテキスト管理  │   │
│                 │  ・アーキテクチャ制約  │   │
│                 │  ・ガードレール     │   │
│                 └────────────────┘   │
└───────────────────────────────────────┘
```

LLM だけでは「何でもする」エージェントになる。
ハーネスで「**何をしてよいか**」を設計する。

---

## GitHub Copilot のハーネス構成

| レイヤー | ファイル/ツール | 役割 |
|---------|-------------|------|
| **Instructions** | `.github/instructions/*.md` | 常時適用されるルール |
| **Skills** | `.github/skills/*/SKILL.md` | 特定タスクの手順書 |
| **Hooks** | `.github/hooks/*.json` | ツール実行の制御 |
| **SDK** | `@github/copilot-sdk` | プログラムからの制御 |

---

## ハーネスエンジニアリングの 3 本柱

1. **コンテキスト管理**
   必要な情報をエージェントに与え、不要な情報を与えない

2. **アーキテクチャ制約**
   `allowedTools` で使えるツールを限定する

3. **品質ゲート**
   `SkillTestHarness` で「意図通りに動くか」を自動検証

# 2. Playwright CLI によるWebアプリ分析

## playwright-cli とは

Playwright チームが**コーディングエージェント向けに設計した CLI ツール**。

- トークン効率を最重視したコマンド体系
- GitHub Copilot・Claude Code から直接呼び出せる
- `playwright-cli install --skills` で Copilot 用 Skill が自動生成される(この作業は済)

---

## 基本コマンド

ユーザーがこれをつかってもいいがこれらのコマンドなどを使用してGHCがWebアプリを操作することができる。

```bash
# ブラウザでページを開く
playwright-cli open https://demo.playwright.dev/todomvc/ --headed

# ページ構造のスナップショット取得
playwright-cli snapshot

# スクリーンショット保存
playwright-cli screenshot --filename=temp/todomvc-initial.png

# ネットワーク通信の確認（APIエンドポイント発見）
playwright-cli network
```

スナップショットの出力を「エージェントへのコンテキスト」として使用することもできる

---

## データを抽出してみる

```bash
# JavaScript で DOM からデータを取得

    completed: li.classList.contains('completed')
  }))
"

# セッション保存（次回も同じ状態から継続できる）
playwright-cli state-save temp/session.json
```

# 3. テスト用ツールの作成（Copilot SDK・Plan→Tools）

---

## Copilot SDK の Plan→Tools パターン

Copilot SDK はエージェントの「アクティブループ」をプログラムから利用できる:

```
send(prompt)
  │
  ▼ [PLAN]  タスクを計画
  │
  ▼ [TOOL]  playwright-cli 実行
  │
  ▼ [TOOL]  ファイル作成
  │
  ▼ [IDLE]  完了
```

このループを TypeScript/C# で制御する

---

## SkillTestHarness の仕組み

`microsoft/copilot-sdk-samples` が提供するテストフレームワーク:

```typescript
const harness = new SkillTestHarness({
  mode: "mock",  // LLM不使用・決定論的
});

const result = await harness.runTest(skill, {
  prompt: "ToDoデータを収集してください",
  criteria: [
    { type: "output_contains", expected: "playwright-cli" },
    { type: "custom", validate: (out) => !out.response.includes("npm install") },
  ]
});
```

---

## 2 つのモードの違い

| | mock モード | live モード |
|---|---|---|
| LLM 呼び出し | **なし** | あり |
| 再現性 | **100%** | 変動あり |
| API キー | **不要** | 必要 |
| CI/CD | **そのまま動く** | 認証設定が必要 |
| 用途 | 構造チェック | 品質確認 |

**まず mock でガードレールを確認し、live で動作品質を確認する**

---

## テスト用ツールを作成する

Copilot に依頼:

```markdown

microsoft/copilot-sdk-samples の SkillTestHarness パターンを参考にして、
mock モードと live モードの両方で実行できるツールを構築してください。

必要なパッケージ:
- @github/copilot-sdk

```

## 受け入れ基準（AcceptanceCriteria）を理解する

4 種類の基準を使い分ける:

```typescript
// 1. TypeScript 構造の存在確認
{ type: "code_compiles" }

// 2. 文字列・正規表現マッチ
{ type: "output_contains", expected: "playwright-cli" }

// 3. テストコードの存在確認
{ type: "test_passes" }

// 4. 任意ロジック（最も強力）
{ type: "custom", validate: (out) => !out.response.includes("rm -rf") }
```

## allowedTools の効果

`allowed-tools` フィールドで**使用ツールを明示的に限定**する:

```yaml
# これのみ許可
allowed-tools: playwright-cli

# 複数許可する場合
allowed-tools: playwright-cli read_file
```

これが「アーキテクチャ制約」の具体的な実装

---

## 作成するアプリの仕様

```
src/DashboardApi/
  ├── Program.cs          ← Minimal API エントリポイント
  ├── Models/
  │   └── TodoItem.cs     ← データモデル

      └── dashboard.db    ← SQLite データストア
```

**API エンドポイント**:

```
GET  /          → ダッシュボード HTML
POST /api/import-csv  → CSV インポート
GET  /api/todos       → 保存済みデータ取得
```

---


# 7. Copilot SDK バッチアプリの構築

---

## バッチアプリの全体設計

2 つの Skill を順番に実行するバッチ処理:

```
[BatchApp]
    │
    ├─ Session 1: web-data-collector Skill
    │      playwright-cli で ToDoMVC 分析
    │      → temp/collected-data.csv に保存
    │
    └─ Session 2: csv-importer Skill
           CSV を POST /api/import-csv でインポート
           → ダッシュボードにデータ反映
```

---

## Copilot SDK の基本パターン（TypeScript）

```typescript
import { CopilotClient } from "@github/copilot-sdk";

const client = new CopilotClient();
await client.start();

// Skill を指定してセッション作成
const session = await client.createSession({
  model: "claude-sonnet-4.6",
  skillDirectories: [".github/skills/web-data-collector"],
});

// イベントを受け取りながらプロンプト送信
session.on((event) => {
  if (event.type === "assistant.message") process.stdout.write(event.data.content);
  if (event.type === "session.idle") console.log("\n完了");
});

await session.send({ prompt: "ToDoMVC からデータを収集してください" });
```

---

## バッチアプリを作成する

Copilot に依頼:

```markdown
src/BatchApp に TypeScript のバッチアプリを作成してください。

要件:
- @github/copilot-sdk を使用する
- web-data-collector Skill でデータ収集（Session 1）
- csv-importer Skill でインポート（Session 2）
- 各セッションは独立して作成・破棄する
- 実行ログを標準出力に出力する
- 環境変数 BATCH_MODEL でモデルを切り替えられる

実行コマンド: npx ts-node src/BatchApp/index.ts
```

---

## Skill 分離の重要性

```typescript
// ✅ 良い設計：Skill ごとにセッションを分離
const collectSession = await client.createSession({
  skillDirectories: [".github/skills/web-data-collector"],  // playwright-cli のみ
});
// ...収集処理...
await collectSession.destroy();

const importSession = await client.createSession({
  skillDirectories: [".github/skills/csv-importer"],  // shell のみ
});
// ...インポート処理...
await importSession.destroy();
```
