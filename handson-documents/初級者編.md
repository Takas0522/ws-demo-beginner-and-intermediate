# 2. 基本操作と最初の会話と設定

## GitHub Copilotの基本操作

<!--画面上での使い方の説明。図表などあればなおよし-->

### GitHub Copilotの会話のモード

- model: 使用するAIモデルを選択するモード
- Ask: 質問や指示を入力して、GitHub Copilotから回答やコードの提案を得るモード
- Agent: GitHub Copilotが自動的にタスクを実行するモード
- Plan: GitHub Copilotがタスクの計画を立てるモード

### 最初の会話

`model`: GPT-5 mini

1. GitHub Copilotを起動する
2. 「Ask」モードで質問や指示を入力する
   - 「Webで現在の東京の天気を確認できますか」
3. 「Agent」モードでGitHub Copilotがタスクを実行するのを待つ
   - 「確認した天気の情報などを要約して `temp/weather_summary.txt` に保存してください」

## GitHub Copilot CLIのインストールと利用

### GitHub Copilot CLIのインストール

GitHub Copilot CLI はスタンドアロンのツール (`copilot` コマンド) として提供されています。以下のいずれかの方法でインストールできます。

**npm（全プラットフォーム）** — 前提: Node.js 22以降
```bash
npm install -g @github/copilot
```

**インストールスクリプト（Linux / macOS）**
```bash
curl -fsSL https://gh.io/copilot-install | bash
```

**Homebrew（macOS / Linux）**
```bash
brew install copilot-cli
```

**WinGet（Windows）**
```powershell
winget install GitHub.Copilot
```

初回起動時にGitHubにログインしていない場合は、`/login` コマンドで認証を求められます。

### GitHub Copilot CLIでの会話

GitHub Copilot CLI はターミナルで `copilot` と入力するだけで起動できます。

```bash
copilot
```

タスクを自然言語で伝えると、Copilot が自律的に作業を進めます。

**2つのインタラクティブモード:**
- **ask/executeモード**（デフォルト）: 要求した処理を即座に実行
- **planモード**: `Shift+Tab` で切り替え。実行前に構造化された計画を表示してから進める

**一発プロンプトで実行（プログラム・インターフェイス）:**
```bash
copilot -p "今週のコミットを一覧してまとめて"
```

> **ヒント**: ファイル変更やコマンド実行は初回利用時に承認を求められます。`--allow-all-tools` で承認をスキップできますが、実行内容は常に確認してください。

## Pythonのインストール

環境競合を避けるため、このハンズオンでは uv ベースでPython環境を構築します。
GitHub CopilotのChatを活用しながら、以下の流れで進めます。

Chatに以下のように質問してみましょう。

> 「uv を使ってこの環境にPythonをセットアップしてください」

次に uv でPythonと仮想環境を準備します。

```bash
uv python install 3.12
uv venv
```

仮想環境の有効化:

- **Windows (PowerShell)**: `.venv\Scripts\Activate.ps1`
- **Linux / macOS**: `source .venv/bin/activate`

依存関係の追加:

```bash
uv pip install requests
```

GitHub Copilotは使用中のOSを認識し、手順の微調整も提案できます。

インストール後は `uv --version` と `python --version` で確認します。

## Agent Browserモードの設定

Agent BrowserモードはGitHub CopilotがブラウザをPlaywrightで操作できるようにする機能です。

GitHub Copilotのチャットで以下のように質問しながら設定を進めます。

> 「Agent Browserモードを有効にする方法を教えてください」

設定の概要:

1. VS Codeの設定 (`Ctrl+,` / `Cmd+,`) を開く
2. `github.copilot.chat.agent.browser.enabled` を検索
3. チェックボックスをオンにして有効化
4. AgentモードのチャットでPlaywrightを使ったブラウザ操作タスクを依頼できるようになる

> **注意**: 初回利用時に必要なパッケージのインストールを求められる場合があります。

## NES(Next Edit Suggestion)の設定

NES (Next Edit Suggestion) は、直前の編集内容に基づいて次に編集すべき箇所を自動で提案する機能です。

GitHub Copilotのチャットで以下のように質問しながら設定できます。

> 「Next Edit Suggestionを有効にする方法を教えてください」

設定手順:

1. VS Codeの設定 (`Ctrl+,` / `Cmd+,`) を開く
2. `github.copilot.nextEditSuggestions.enabled` を検索
3. `true` に設定して有効化
4. コードを編集すると、矢印アイコンで次の変更候補が表示される
   - `Tab` キーで提案を受け入れる
   - `Esc` キーでスキップする

# 4. GitHub Copilotのカスタマイズ

## Custom Instructions

Custom Instructionsは、GitHub Copilotの回答スタイルやコーディング規約をリポジトリ全体に適用するための設定機能です。
`.github/copilot-instructions.md` ファイルに記述することで、プロジェクト固有のコミュニケーションルールや技術スタックの前提知識をCopilotに共有できます。

**主な用途:**
- コーディング規約の共有（インデント、命名規則など）
- 使用する言語・フレームワークの明示
- 回答言語やトーンの指定

### 利用

1. `.github/copilot-instructions.md` を作成する
2. プロジェクト固有のルールや前提知識を記述する

   例:
   ```markdown
   - Python 3.12を使用する
   - 回答は常に日本語で返す
   - コメントは日本語で記述する
   - テストはpytestを使用する
   ```
3. 以降、全てのGitHub Copilotの会話で自動的に適用される

## Agent Skillsの利用

Agent Skillsは、指示・スクリプト・補助リソースをひとまとめにして再利用できる機能です。
VS Code / Copilot CLI / Copilot coding agent で共通利用できるオープン標準です。

**主な用途:**
- 繰り返し行うレビューやドキュメント作成の手順をテンプレート化
- チーム共通の作業フローを再利用可能な形にまとめる
- 特定のタスクに特化した指示のパッケージ化

### 利用

1. `.github/skills/` 配下にスキル用ディレクトリを作成する（例: `.github/skills/web-testing/`）
2. ディレクトリ内に `SKILL.md` を作成し、YAMLフロントマター（`name`, `description`）と手順を記述する
3. 必要に応じて同じディレクトリにスクリプトやサンプルを配置する
4. Chatで `/スキル名` で呼び出す（または関連タスク時に自動適用される）

## Custom Agentsの利用

Custom Agentsは、特定のタスク専用のエージェントを定義しておく機能です。
`.agent.md` ファイルにエージェント定義を記述し、Chatのモードセレクタから呼び出せます。

**主な用途:**
- レビュー専用エージェント、テスト専用エージェントなどロール別のエージェントを定義
- 使用するツールや制約をエージェントごとにカスタマイズ
- 共通のコンテキストや指示を持つ専用エージェントの共有

### 利用

1. `.github/agents/` ディレクトリに `*.agent.md` ファイルを作成する
2. YAMLフロントマターでエージェント名や説明、利用可能なツールを定義する
3. Chatのモードセレクタで作成したエージェントを選択して利用する

## カスタマイズの作り方

各カスタマイズ機能は VS Code内の GUIからも作成できます。
Chatビューの Configure Chat（歯車アイコン）から Chat Customizations editor を開き、
Instructions / Prompts / Skills / Agents タブで作成できます。

コマンドパレットから作る場合:

- `Chat: Open Chat Customizations`
- `Chat: New Instructions File`
- `Chat: New Prompt File`
- `Chat: New Custom Agent`

AIでひな形を生成する場合:

- `/init`（`.github/copilot-instructions.md` の生成）
- `/create-instruction` / `/create-prompt` / `/create-skill` / `/create-agent`

GitHub Copilotで作成を依頼するだけでも構いません。

# 5. CLIでのGitHub Copilotの活用


## CLIでの会話のモード

GitHub Copilot CLI では以下の2つのインタラクティブモードで作業を進められます。

| モード | 切り替え方 | 概要 |
|------|-----------|------|
| ask/execute モード（デフォルト） | 起動直後の状態 | タスクを伝えると即座に実行する |
| plan モード | `Shift+Tab` で切り替え | 実行前に構造化された計画を立ててから進める |

**使い分けの目安:**
- すぐに実行してほしい → ask/execute モード（デフォルト）
- 複雑なタスクで事前に計画を確認したい → plan モード
- スクリプトから一発実行したい → `copilot -p "プロンプト"` (プログラム・インターフェイス)

## 便利なビルドインの利用

`agent-customization` はVS CodeのGitHub Copilotに組み込まれたビルトインスキルです。
Custom Instructions / Agent Skills / Custom Agents などのカスタマイズファイルの作成や編集をサポートします。

**主な機能:**
- `.instructions.md` / `.prompt.md` / `SKILL.md` / `.agent.md` ファイルの作成・編集
- YAMLフロントマターの構文サポート
- `applyTo` パターンの設定ガイダンス

**Chatでの利用:**
カスタマイズファイルの作成・編集を依頼するだけで自動的に適用されます。例:
```
Custom Instructionsのファイルを作成してください
Agent Skillのひな形を生成してください
```

# 6. GitHub Copilotでのブラウザ操作作業

## Webアプリケーションの構造分析

Agent Browserモードを使用して、GitHub Copilotに実際のWebサイトを開かせて構造を分析してもらいます。

例: GitHubのトップページの構造分析

```
「Agentモード」で指示:
https://github.com を開き、トップページのメインナビゲーション
メニューの構造を分析してまとめてください
```

## 作業のPrompt File化

ブラウザ操作の手順を再利用可能なPrompt Fileとして保存します。

1. Agentモードで作業を実行し、手順を確認する
2. 手順を `.github/prompts/タスク名.prompt.md` に保存する
3. 次回以降は Chatで `/<タスク名>` スラッシュコマンドで呼び出して再利用する

# 7. 業務での活用

## Excel/Word/PowerPointの作成

GitHub Copilotの Agentモードを使って、業務書類の作成作業を支援できます。

例: 会議資料のドラフト作成

```
「当月の売上対比データを軽くまとめたcsvDataから、
PowerPointのスライド構成ドラフトをMarkdown形式で作成してください」
```

### 既存資料の分析

既存の資料（PDF、Markdown、テキスト）をChatに読み込ませて、要約・分析・追記事項の抜き出しを依頼できます。

```
「添付の議事録から次のアクションアイテムと担当者を一覧表にまとめてください」
```

### Prompt Fileによる再利用

業務で繰り返す作業手順をPrompt Fileとして保存することで、次回からは一言で実行できるようになります。

1. 現在の手順を `.github/prompts/業務名.prompt.md` に保存
2. 必要な標準出力形式、例外処理、テンプレートをプロンプト内に記述する
3. チーム内で共有して再利用を促進する

### 利用

```
/<Prompt File名> 「作業の内容」
```
