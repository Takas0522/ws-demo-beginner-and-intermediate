# App2: 開発状況確認ダッシュボード 機能要件

## 1. 概要

App1（サービスダッシュボード）で扱う各電子サービスの開発プロジェクトに関して、チケット・PR管理から工数・原価管理まで一元的に可視化する開発状況確認ダッシュボードです。事業部・部署・プロジェクト単位での進捗を把握し、開発リソースのコスト管理も行います。

---

## 2. ドメインモデル概要

```
事業部 (BusinessUnit)
└── サービス (Service)           ← App1と対応
    └── プロジェクト (Project)
        ├── スプリント (Sprint)
        ├── チケット (Ticket)
        │   ├── タスク (Task)
        │   └── コメント (Comment)
        └── プルリクエスト (PullRequest)
            └── レビュー (Review)

部署 (Department)
└── メンバー (Member)
    └── 工数実績 (WorkLog)
```

## 3. 機能要件

### 3.1 ダッシュボード表示

#### 3.1.1 全社開発サマリービュー

| 表示項目 | 説明 |
|----------|------|
| 進行中プロジェクト数 | ステータス別内訳付き |
| 全体タスク数 | 未着手 / 進行中 / 完了 / ブロック別 |
| 今週完了タスク数 | - |
| オープンPR数 | 全プロジェクト合計 |
| 総実績工数（当月） | 全プロジェクト合算 |
| 総原価（当月） | 全プロジェクト合算 |
| 予算消化率 | 実績原価 / 予算原価 × 100 |

#### 3.1.2 事業部別ビュー

| 表示項目 | 説明 |
|----------|------|
| 事業部名 | - |

| 進捗サマリー | 完了タスク数 / 全タスク数 |
| 実績工数合計 | 当月 |
| 原価合計 | 当月 |

#### 3.1.3 プロジェクト別詳細ビュー

| 表示項目 | 説明 |
|----------|------|
| プロジェクト名・概要 | 対象サービス名・開始日・終了予定日 |
| 進捗率 | 完了タスク数 / 全タスク数（スプリント別も表示） |
| タスク数サマリー | 未着手 / 進行中 / 完了 / ブロック |
| バグ起票数 | 重要度別（Critical / High / Medium / Low） |
| オープンPR数 | レビュー待ち / 承認済み / 変更要求 |
| PRマージ所要時間（平均） | オープンからマージまでの経過時間 |
| 実績工数 | 当月・累計（スプリント別グラフ） |
| 予定残り工数 | 完了見込み計算 |
| 実績原価 | 当月・累計（工数 × 単価） |
| 予定残り原価 | 残り工数 × 平均単価 |
| EVM（アーンドバリュー管理） | PV / EV / AC / CPI / SPI |
| スプリントバーンダウン | チャート表示 |
| スプリントベロシティ | 直近スプリントの推移グラフ |

#### 3.1.4 チケット一覧・詳細ビュー

| 表示項目 | 説明 |
|----------|------|
| チケットID・タイトル | - |

| 実績工数 | ログ済み工数の合計 |
| 開始日・期限日 | - |
| 関連PR | リンク付き |

#### 3.1.5 PR（プルリクエスト）一覧・詳細ビュー

| 表示項目 | 説明 |
|----------|------|
| PR番号・タイトル | - |
| ステータス | open / merged / closed |
| 作成者・担当レビュアー | - |
| 対象ブランチ | base / head |
| 変更ファイル数・行数 | 追加 / 削除 |
| レビューステータス | 承認数 / 変更要求数 |
| オープンからの経過時間 | - |
| 関連チケット | リンク付き |

### 3.3 CSV エクスポート

#### エクスポート対象

| エクスポート種別 | 含まれる主なカラム |
|-----------------|-------------------|
| プロジェクト進捗サマリー | プロジェクトID/名, 事業部, 対象サービス, 開始日, 終了予定日, 全タスク数, 完了タスク数, 進捗率, 実績工数, 予定残り工数, 実績原価, 予定残り原価, バジェット, 予算消化率 |
| タスク詳細 | タスクID, プロジェクトID/名, スプリント名, タイトル, 種別, 優先度, ステータス, 担当者, 部署, 見積もり工数, 実績工数, 開始日, 期限日, 完了日 |
| PR詳細 | PR番号, プロジェクトID/名, タイトル, ステータス, 作成者, レビュアー, 変更ファイル数, 追加行数, 削除行数, オープン日時, マージ日時, 所要時間(h) |
| 工数・原価明細 | メンバーID/名, 部署, プロジェクトID/名, タスクID, 作業日, 実績工数(h), 単価, 原価, 作業内容 |

#### エクスポート要件

- 文字コード: UTF-8 with BOM（Excel対応）
- 日付フォーマット: YYYY-MM-DD
- 数値フォーマット: カンマなし
- ダウンロード方式: ブラウザダウンロード（ファイル名に期間・種別を含む）

## 4. 工数・原価計算ルール

| 計算項目 | 計算式 |
|----------|--------|
| 実績原価 | Σ（実績工数 × メンバー単価） |
| 予定残り原価 | Σ（残り予定工数 × メンバー単価） |
| CPI（コスト効率指数） | EV / AC |
| SPI（スケジュール効率指数） | EV / PV |
| EAC（完了時コスト予測） | BAC / CPI |
| VAC（コスト差異） | BAC - EAC |

メンバー単価はデフォルト値または部署ごとの設定値を使用します（個人情報保護の観点から実名単価は管理者のみ参照可）。

## 6. APIエンドポイント概要（Backend）

| メソッド | パス | 説明 |
|----------|------|------|
| GET | /api/summary | 全社開発サマリー取得 |
| GET | /api/business-units | 事業部一覧取得 |
| GET | /api/business-units/{id}/summary | 事業部別開発サマリー取得 |
| GET | /api/departments | 部署一覧取得 |
| GET | /api/projects | プロジェクト一覧取得（絞り込み対応） |
| GET | /api/projects/{id} | プロジェクト詳細取得 |
| GET | /api/projects/{id}/sprints | スプリント一覧取得 |
| GET | /api/projects/{id}/tickets | チケット一覧取得 |
| GET | /api/projects/{id}/pull-requests | PR一覧取得 |
| GET | /api/tickets/{id} | チケット詳細取得 |
| GET | /api/pull-requests/{id} | PR詳細取得 |
| GET | /api/members | メンバー一覧取得 |
| GET | /api/members/{id}/work-logs | 工数ログ取得 |
| GET | /api/export/project-summary | プロジェクトサマリーCSVエクスポート |
| GET | /api/export/tasks | タスク詳細CSVエクスポート |
| GET | /api/export/pull-requests | PR詳細CSVエクスポート |
| GET | /api/export/work-cost | 工数・原価明細CSVエクスポート |

クエリパラメータ共通: `from`, `to`（日付範囲）, `businessUnitIds`, `departmentIds`, `projectIds`, `memberIds`

## 7. 起動情報

| レイヤー | ポート | 備考 |
|----------|--------|------|
| Frontend (React) | 3002 | Vite dev server / 本番はNginx等 |
| Backend (ASP.NET Core) | 5002 | HTTP |
| Database (PostgreSQL) | 5434 | ホスト側ポート（コンテナ内5432） |

個別起動コマンド:

```bash
# Frontend
cd src/app2-dev-dashboard/frontend && npm run dev

# Backend
cd src/app2-dev-dashboard/backend && dotnet run

# Database (Docker)
docker compose -f src/app2-dev-dashboard/database/docker-compose.yml up -d
```
